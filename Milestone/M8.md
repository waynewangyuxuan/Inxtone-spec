# Milestone 8: Polish & v0.1.0 Release

- **Status**: Draft
- **Target Date**: TBD (after M7)

## Goal

Make Inxtone production-ready for v0.1.0 release. Semantic search upgrades discoverability, UI polish and i18n complete the user experience, and documentation + CLI prepare the product for public release.

## Success Criteria

```
Track A: Semantic search functional, seed data migrated to TS, hybrid ranking working
Track B: i18n infrastructure, entity links, version history, UI polish complete
Track C: CLI complete, documentation written, v0.1.0 tagged
All: Performance targets met, all tests pass
```

## Technical Notes

### Performance Targets

| Metric | Target |
|--------|--------|
| Startup time | < 500ms |
| Idle memory | < 100MB |
| Search latency (FTS5) | < 100ms |
| Search latency (hybrid) | < 300ms |
| Chapter load time | < 200ms |

### Worktree Setup

```bash
git worktree add ../inxtone-m8a feat/m8-track-a-search
git worktree add ../inxtone-m8b feat/m8-track-b-ui-polish
git worktree add ../inxtone-m8c feat/m8-track-c-release
```

Tracks A and B are independent. Track C (docs/release) starts mid-milestone after A and B stabilize.
Merge order: A -> B -> C -> final integration on main -> tag v0.1.0.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| sqlite-vec compatibility issues | Medium | High | Test early; fallback to FTS5-only |
| i18n scope creep (200+ strings) | Medium | Medium | Phase by page, defer CLI/server to follow-up |
| PDF export dependency size | Medium | Low | Use pdf-lib (lightweight), defer puppeteer |
| Release blocking issues | Medium | Medium | Buffer final 2 days for hotfixes |
