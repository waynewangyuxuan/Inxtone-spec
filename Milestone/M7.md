# Milestone 7: Quality & Visualizations

- **Status**: Draft
- **Target Date**: TBD (after M6)
- **Owner**: Wayne
- **Duration**: 2 weeks
- **Dependencies**: M6 (Smart Intake) — Complete
- **Structure**: 3 parallel tracks (A/B/C) for git worktree development

## Goal

Give writers the ability to **see and verify their story's structural integrity**. The QualityService catches consistency issues automatically, while visualizations (relationship map, timeline, pacing) make story structure tangible.

## Success Criteria

```
Track A: QualityService runs 26 rules, API routes functional, layered context injection working
Track B: Relationship map, timeline, pacing visualizations all render with demo data
Track C: Quality panel UI integrated, CLI check command, pre-export checks wired
All: 3 tracks merge cleanly, tests pass
```

## Technical Notes

### Visualization Libraries

| Visualization | Library | Rationale |
|---------------|---------|-----------|
| Relationship Map | `d3-force` + React SVG | Fine-grained control, no heavy deps |
| Timeline | Custom React + CSS | Simple enough, avoids lib overhead |
| Pacing Charts | `recharts` | Lightweight React charts |

### Worktree Setup

```bash
git worktree add ../inxtone-m7a feat/m7-track-a-quality-core
git worktree add ../inxtone-m7b feat/m7-track-b-visualizations
git worktree add ../inxtone-m7c feat/m7-track-c-quality-ui
```

Track B can develop independently. Track C depends on Track A API.
